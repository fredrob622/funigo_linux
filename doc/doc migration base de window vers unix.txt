2. Configuration pour les caractères japonais (UTF-8)
bash# Édition de la configuration MySQL
sudo nano /etc/mysql/mariadb.conf.d/50-server.cnf

Ajoutez ces lignes dans la section [mysqld] :
[mariabd]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci

sudo vim /etc/mysql/mariadb.conf.d/50-client.cnf

[client-mariadb]
default-character-set=utf8mb4

4. Export de la base depuis Windows 11
Sur votre Windows 11, ouvrez PowerShell ou CMD :

Export avec mysqldump (en spécifiant UTF-8)
mysqldump -u japadmin -p --default-character-set=utf8mb4 --databases jap > jap_export.sql


PS C:\Users\fredr> mysqldump -u japadmin -p --default-character-set=utf8mb4 --databases jap > jap_export.sql
Enter password: ********

*********** Modifier japadmin'@'localhost

GRANT PROCESS, SELECT, LOCK TABLES, RELOAD ON *.* TO 'japadmin'@'localhost';

pb ERROR jap_utf8.sql: affix definition for MySpell/Hunspell, with BOM, 1st line "-- MySQL dump 10.13  Distrib 8.0.41, for Win64 (x86_64)"
Le fichier a toujours un problème - le BOM (Byte Order Mark) interfère. Supprimons-le :

# Supprimer le BOM UTF-8
sed '1s/^\xEF\xBB\xBF//' jap_utf8.sql > jap_clean.sql

# Vérification
file jap_clean.sql

jap_clean.sql: Unicode text, UTF-8 text, with very long lines (23175), with CRLF line terminators

**************************************************************************************************************************************
Pb ERROR 1273 (HY000) at line 25: Unknown collation: 'utf8mb4_ja_0900_as_cs'

Le problème est que votre MySQL sur Raspberry Pi ne reconnaît pas la collation utf8mb4_ja_0900_as_cs (spécifique à MySQL 8.0 sur Windows). 
Il faut la remplacer par une collation compatible.

fred@funigo:~/Documents $ sed -i 's/utf8mb4_ja_0900_as_cs/utf8mb4_unicode_ci/g' jap_clean.sql
