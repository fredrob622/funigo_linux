Le fichier a toujours un problème - le BOM (Byte Order Mark) interfère. Supprimons-le :

# Supprimer le BOM UTF-8
sed '1s/^\xEF\xBB\xBF//' jap_utf8.sql > jap_clean.sql

# Vérification
file jap_clean.sql

**************************************************************************************************************************************
Pb ERROR 1273 (HY000) at line 25: Unknown collation: 'utf8mb4_ja_0900_as_cs'

Le problème est que votre MySQL sur Raspberry Pi ne reconnaît pas la collation utf8mb4_ja_0900_as_cs (spécifique à MySQL 8.0 sur Windows). 
Il faut la remplacer par une collation compatible.

fred@funigo:~/Documents $ sed -i 's/utf8mb4_ja_0900_as_cs/utf8mb4_unicode_ci/g' jap_clean.sql
